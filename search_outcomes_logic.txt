How the 3 outcomes work

1) Exact match shown (with "Exact-match recommendations")
- The backend first checks for products that match all 4 inputs exactly:
  room_size + room_type + style + budget.
- If found, it returns only those products, and each gets match_score = 100.

2) "No products found..." message but still shows recommendations
- This happens when no exact 4-field match exists, so fallback starts.
- In fallback, room_type + style + budget stay strict, but room_size is relaxed.
- So it says "no products for your selected size," but still shows closest items with the same room_type/style/budget.

3) "No products found..." and no suggestions at all
- This means fallback also failed.
- So there are zero products in DB for that room_type + style + budget combination (across any room size), therefore the suggestions list is empty.

Key rule in the logic
- style and budget are never relaxed; only room_size can be relaxed in fallback.
- That is why case 2 can show alternatives, while case 3 cannot.
